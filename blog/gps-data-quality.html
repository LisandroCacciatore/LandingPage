Data Quality

Por qué tus datos de GPS son basura (y cómo limpiarlos)

Oct 12, 2024 • 6 min lectura

Un error común al importar CSVs de Catapult es ignorar los valores nulos. El problema no es que falten datos. Es que nadie valida el schema antes de analizar.

El problema silencioso

Tu software de GPS exporta un CSV. Lo abrís en Excel o lo importás directo a tu dashboard. Todo parece funcionar. Pero hay un detalle que la mayoría ignora: los valores faltantes no siempre aparecen como celdas vacías.

Dependiendo de cómo esté configurado el export, un jugador que no entrenó puede aparecer como:

  • NULL (correcto, pero raro en CSVs)
  • Celda vacía (depende del software)
  • 0 (el más peligroso)
  • N/A o - (inconsistente entre exports)

Por qué importa

Imaginá que estás calculando la carga crónica promedio de las últimas 4 semanas. Si un jugador estuvo lesionado 2 semanas y su software pone 0 en vez de NULL, tu promedio está diluyendo su carga real.

"Un cero no es lo mismo que un dato faltante. Uno dice 'no entrenó'. El otro dice 'entrenó pero sin carga', lo cual es imposible."

Validación básica del schema

1. Identificar tipos de columnas

Antes de importar, revisá que cada columna tenga el tipo de dato correcto:

  • Player Name / ID: Texto (String)
  • Date: Fecha (Date) — no texto
  • Total Distance, HSR, Sprints: Números (Float/Integer)
  • Session Type: Categoría (Enum o String limitado)

2. Detectar valores sospechosos

Un script simple en Python puede detectar problemas:

import pandas as pd

df = pd.read_csv('catapult_export.csv')

# Detectar ceros sospechosos
suspicious_zeros = df[(df['Total Distance'] == 0) & (df['Duration'] > 0)]
print(f"Sesiones con distancia 0 pero duración > 0: {len(suspicious_zeros)}")

# Detectar valores nulos no marcados
null_as_text = df[df['Player Name'].str.contains('null|NULL|N/A', na=False)]
print(f"Valores nulos mal formateados: {len(null_as_text)}")

3. Normalizar antes de analizar

Una vez detectados los problemas, normalizá los datos antes de pasarlos a tu dashboard o análisis: 

# Convertir ceros sospechosos a NaN (pandas trata esto correctamente)
df.loc[df['Total Distance'] == 0, 'Total Distance'] = pd.NA

# Estandarizar nombres de jugadores
df['Player Name'] = df['Player Name'].str.strip().str.title()

Checklist de validación diaria

Antes de confiar en tus datos GPS, verificá:

  1. ✅ Que no haya jugadores con Total Distance = 0 y duración > 0
  2. ✅ Que las fechas estén en formato consistente (YYYY-MM-DD)
  3. ✅ Que no haya nombres de jugadores duplicados con spelling diferente
  4. ✅ Que las métricas críticas (HSR, Sprints) no tengan outliers sin explicación
  5. ✅ Que el rango de valores sea razonable para tu deporte y nivel
