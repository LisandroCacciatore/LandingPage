Por qué tus datos de GPS son basura (y cómo limpiarlos)

12 Oct, 2024 • 8 min lectura

Importar un CSV parece fácil hasta que intentas calcular el promedio de carga crónica y te das cuenta de que el 30% de tus datos son ceros que debían ser nulos.

El problema de los "Ceros Falsos"

En la mayoría de los software de GPS deportivos, cuando un jugador no entrena, el sistema a veces exporta un 0 en lugar de un null o celda vacía. Para una base de datos, 0 es un número. Es un dato.

"Si promedias la carga de un jugador incluyendo los días que estuvo enfermo (0 km), estás subestimando su capacidad real y arriesgando su vuelta al campo."

Cómo detectarlo con Python

Aquí tienes un script simple usando Pandas para auditar tu CSV antes de subirlo a BigQuery:

import pandas as pd

# Cargar datos
df = pd.read_csv('export_gps.csv')

# Verificar ceros en columnas clave
zeros = df[df['Total Distance'] == 0].count()

print(f"Filas con distancia cero: {zeros}")

La solución arquitectónica

No limpies datos en Excel manualmente. Diseña un pipeline de ingestión que:

  • Detecte jugadores que no marcaron sesión.
  • Transforme 0 a null si no hubo sesión activa.
  • Genere una alerta si la calidad del dato baja del 95%.
